# README — Preservação, Consolidação e Reentrada

## Estatuto
Documento **canónico de orientação**.

Este README explica **como usar corretamente o ChatGPT como laboratório**, como preservar trabalho relevante fora do canvas e como **retomar contexto sem perda nem caos**.

Não é normativo no sentido operacional, mas é **vinculativo enquanto orientação do regime Grilo Falante UNI**.

---

## Princípio fundamental

> **O ChatGPT é um laboratório cognitivo, não um arquivo.**

Tudo o que permanece apenas no canvas é **transitório por definição**.

---

## Estados possíveis do trabalho

Todo o conteúdo produzido deve ser entendido como estando num destes estados:

1. **Exploratório**
   - Ideias em formação.
   - Conversas livres.
   - Pode ser descartado sem remorso.

2. **Consolidado**
   - Material estabilizado num chat agregador.
   - Pode originar patches ou documentos canónicos.

3. **Promovido**
   - Exportado explicitamente para fora do ChatGPT.
   - Versionado e preservado.
   - O repositório externo é a fonte de verdade.

Misturar estes estados conduz a caos.

---

## Ciclo recomendado (uso saudável)

1. **Explorar**
   - Trabalhar em vários chats.
   - Não tentar preservar tudo.

2. **Consolidar**
   - Criar um chat dedicado por tema.
   - Integrar apenas o que resistiu.

3. **Auditar**
   - Questionar fluência excessiva.
   - Confirmar limites e pressupostos.

4. **Promover**
   - Exportar documentos estáveis.
   - Atribuir versão.

5. **Preservar**
   - Usar Git, filesystem local, backups normais.

6. **Reentrar**
   - Voltar a explorar sem carregar o passado.

---

## Promoção explícita (regra crítica)

Nada é considerado preservado enquanto não houver:

- decisão humana explícita de promoção;
- exportação para fora do ChatGPT;
- local externo controlado pelo utilizador.

Automatismos silenciosos são rejeitados.

---

## Reentrada cognitiva

Ao regressar a um trabalho:

- **não confiar na memória do chat**;
- usar documentos promovidos como âncora;
- tratar o canvas apenas como apoio temporário.

Se não houver documento promovido, o trabalho deve ser tratado como exploratório.

---

## Sinais de alerta

Parar e consolidar quando:
- já não sabes onde está a versão certa;
- sentes necessidade de "exportar tudo";
- existem demasiados documentos no canvas.

Estes sinais indicam **perda de regulação**, não falta de ferramentas.

---

## Relação com o Documento 08

Este README é uma aplicação direta do fundamento regulatório:

- produção simbólica ilimitada sem promoção → colapso;
- regulação explícita → sobrevivência do trabalho.

---

## O que este README não faz

- Não automatiza exportações.
- Não substitui responsabilidade humana.
- Não promete recuperação perfeita de contexto.

---

## Em caso de dúvida

> **Explora menos. Consolida mais cedo. Promove explicitamente.**

Esquecer faz parte da regulação.
