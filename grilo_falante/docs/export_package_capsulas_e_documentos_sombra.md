# Export Package — Cápsulas, Documentos Sombra e Artefactos Canónicos

## Finalidade
Este documento define **tudo o que é necessário** para exportar o regime atual (pipeline v3 + grafo v2 + Objeto Digital) para **outro chat**, sem perda de contexto, autoridade ou coerência ontológica.

Funciona como **índice-mestre** e checklist de exportação.

---

## 1. Cápsulas Conceptuais (invariantes)

Estas cápsulas fixam ideias que **não podem voltar a ser redescobertas** noutro chat.

### CAPSULA_01_OBJETO_DIGITAL.md
- Objeto Digital como entidade socio-técnica
- Identidade, fronteiras, autoridade, ciclo de vida
- Distinção objeto vs código vs modelo

### CAPSULA_02_FASE_MINUS_1_LOCK_ONTOLOGICO.md
- FASE −1 como gate absoluto e recorrente
- Condições de reexecução
- Prevenção de deriva semântica

### CAPSULA_03_APRENDIZAGEM_EXTERNA_AO_MODELO.md
- O modelo não aprende
- Aprendizagem delegada ao sistema
- Memória normativa externa

### CAPSULA_04_PIPELINE_INSTRUMENTAL.md
- Pipeline como instrumento, não fonte de significado
- Separação exploração / decisão / promoção

### CAPSULA_05_PROFUNDIDADE_CUMULATIVA_GOVERNADA.md
- Profundidade como propriedade sistémica
- Transversalidade a domínios (código, escrita, jogos)

## 5. Instruções de Exportação para outro chat

Ao iniciar um novo chat:

1. Colar este documento.
2. Declarar: "Importar regime canónico v3".
3. Materializar:
   - todas as cápsulas;
   - documentos canónicos;
   - documentos sombra (opcional, mas recomendado).
4. Executar ACORDAR.
5. Declarar ou carregar OBJECT_DIGITAL.
