# Task: Implement [Feature Name]

## Objective
Add [feature name] functionality to enable [specific user/business value or technical capability].

## Pre-Implementation Checklist (Karpathy Principles)
- [ ] **Think Before Coding** — What is the minimal understanding needed? Can I explain the problem in one sentence? What assumptions am I making?
- [ ] **Simplicity First** — What's the simplest possible solution? Am I over-engineering? Can I remove any component?
- [ ] **Surgical Changes** — Can I make this change without restructuring existing code? What is the exact minimum diff?
- [ ] **Goal-Driven Execution** — How will I verify success? What does "done" look like? What's my rollback plan if this goes wrong?

[Explain the problem this feature solves or opportunity it creates. Reference any related issues, user feedback, or planning documents.]

## Constraints
- Must integrate cleanly with existing architecture
- Should follow established patterns for similar features in the codebase
- Public APIs (if any) must be backward compatible
- Performance impact should be minimal and justified
- Must not introduce breaking changes without explicit versioning strategy
- Security considerations must be addressed

## Success Criteria
- Feature works as specified in all expected scenarios
- Automated tests cover new functionality (unit, integration, as appropriate)
- Manual testing confirms expected behavior
- Documentation updated to reflect new feature
- No regressions in existing functionality
- Performance benchmarks within acceptable ranges
